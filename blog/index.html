探索citra移植至ios的可能性

2020年8月29日
探索citra移植至ios的可能性 一直想有生之年在ios/tvos上运行citra，帧数什么的无所谓，1帧能玩，2帧流畅，3帧电竞！ 简易学习了一下opengl的安装与卸载后，对移植跃跃欲试。 编译 调整一下cmake配置，找个ios toolchain拷过来。找个demo拷贝下ios上初始化gl环境的部分。我不生产代码，我只是代码的搬运工。 cmake .. -DCMAKE_OSX_ARCHITECTURES="arm64" \ -DCMAKE_BUILD_TYPE=Release \ -DCMAKE_TOOLCHAIN_FILE="../toolchain/iOS.cmake" \ -DCMAKE_OSX_DEPLOYMENT_TARGET="9.0" 然后就步步高编译器，哪里报错删哪里。 [ 98%] Built target citra-room [100%] Built target citra_ios [100%] Built target tests 耶，成功编译。 cp citra_ios deb/Applications/citra_ios.app/ find . -name ".DS_Store" -delete dpkg-deb -b deb citra.deb scp -P 2222 citra.deb root@localhost: ssh usb "killall citra_ios; dpkg -i citra.deb; /usr/bin/ldid -P -K/usr/share/substitute/signcert.p12 -S/usr/share/entitlements/inject.xml -M /Applications/citra_ios.app/citra_ios; /usr/local/bin/openBundleId com.brend0n.citraios" 最后写个脚本打包安装运行一条龙。 运行 运行一下看看，果然闪退。调试下看看是什么情况，发现需要至少gles3.1的环境。 if (!(GLAD_GL_VERSION_3_3 || GLAD_GL_ES_VERSION_3_1)) { return VideoCore::ResultStatus::ErrorBelowGL33; } 那么如果硬要继续运行呢？当然不行，会在一个不存在的gl函数调用上crash。 一查发现高贵的苹果竟然弃用了opengles，这下等苹果更新3.……
